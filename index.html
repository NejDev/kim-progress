Fase 1 — Backend Core

bem avançado
  • ✓ Category Module Categorias e subcategorias.
  • ✓ Product Module Produtos ativos, slug, categoria e variantes.
  • ✓ Customer Module Cliente com conta, guest e autenticação própria.
  • ✓ Address Module Endereços por cliente e principal.
  • ✓ Admin + JWT Login admin, BCrypt, JWT e rotas protegidas.
  • ✓ Order + OrderItem Pedido, itens, total calculado no backend e respostas completas.

Fase 2 — Pedido real

bem avançado
  • ✓ Estoque Validação antes de adicionar item e baixa automática quando pedido vira PAGO.
  • ✓ Transição de status Fluxo controlado: PENDENTE → PAGO → EM_SEPARACAO → ENVIADO → ENTREGUE, com bloqueio de transições inválidas.
  • ✓ Pedido completo GET /api/orders, GET /api/orders/{id} e GET /api/orders/customer/{customerId} retornam itens do pedido.
  • ✓ Cliente acompanha pedidos Login de cliente com JWT, token CUSTOMER, rotas protegidas e acesso restrito aos próprios pedidos.
  • → Cancelamento Cancelar pedido e, se necessário, devolver estoque.

pendente

pendente

pendente
